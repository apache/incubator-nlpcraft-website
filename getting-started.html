Getting Started

In this section we'll take a quick 10 minutes look at running NLPCraft examples. Instead of building our own model we will use an example model shipped with NLPCraft along with demonstrating the main components of NLPCraft - data probe, REST server, and nlpcraft.{sh|cmd} script.

We assume the following:

  • You downloaded NLPCraft 0.7.4 as a ZIP archive.
  • You followed installation instructions.
  • You are using MacOS/Linux environment.

Data Probe & REST Server

Data probes are used to deploy and host data model, while REST server (or a cluster of servers) is used to accept client REST calls and route them to the data model deployed on data probes.

Data probe and REST server are both Java applications and therefore can be started in many different ways. In this tutorial, we'll use nlpcraft.{sh|cmd} script that comes with NLPCraft and acts as a central management utility responsible for executing most of the routine tasks in NLPCraft in a simple and productive way.

To start REST server and data probe with default configurations - run the following two commands:

Command REST Server Output Data Probe Output 
                    $ bin/nlpcraft.sh start-server
                    $ bin/nlpcraft.sh start-probe

At this point you have both the data probe and the REST server started and connected to each other.

Weather Forecast

We'll be testing Weather Example to ask questions about weather forecast using REST APIs. This example returns a comprehensive JSON weather data for variety of different inquiries about the past, present or future weather conditions.

We will be using NLPCraft CLI script to issue REST calls. Let's start NLPCraft CLI in interactive REPL mode by running nlpcraft.{sh|cmd} script with no argument:

Command Output 
                    $ bin/nlpcraft.sh

NOTES:

  • Note that when you start NLPCraft CLI it automatically detects the REST server and data probe we have started in the previous step.
  • You can see the status of the REST server and data probe in the prompt and you can always use info command to see a full information.
  • When NLPCraft CLI detects running REST server it automatically signs in with the default user account to obtain access token that is required for all REST calls. It will automatically use this access token while in REPL mode whenever required so that you don't have to specify it manually.
  • Use Tab key anytime for command auto-suggestion and auto-completion.
  • You can also use ↑ and ↓ keys to scroll through command history.
