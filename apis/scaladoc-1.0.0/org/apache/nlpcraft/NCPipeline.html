trait NCPipeline

NLP processing pipeline for the input request. Pipeline is associated with the model.

An NLP pipeline is a container for various processing components that take the input text at the beginning of the pipeline and produce the list of [[NCEntity entities]] at the end of the pipeline. Schematically the pipeline looks like this:

+----------+        +-----------+
*=========*    +---------+    +---+-------+  |    +---+-------+   |
:  Text   : -> |  Token  | -> | Token     |  | -> | Token      |  | ----.
:  Input  :    |  Parser |    | Enrichers |--+    | Validators |--+      \
*=========*    +---------+    +-----------+       +------------+          \
}
+-----------+        +----------+        +--------+    /
*=========*    +---+--------+  |    +---+-------+  |    +---+-----+  |   /
:  Entity : <- | Entity     |  | <- | Entity    |  | <- | Entity  |  | <-
:  List   :    | Validators |--+    | Enrichers |--+    | Parsers |--+
*=========*    +------------+       +-----------+       +---------+

[[NCTokenParser]] is responsible for taking the input text and tokenize it into a list of [[NCToken ]]. This process is called tokenization, i.e. the process of demarcating and classifying sections of a string of input characters. There's only one token parser for the pipeline.
