Overview

bin/nlpcraft.{sh|cmd} script is a central management utility for NLPCraft that supports REPL (interactive) and command line modes. Most of the functions this script does can be done through running Java applications or using REST API - yet using this script you can perform the same tasks much simpler and quicker. From starting and managing REST server and probes, running various built-in tools, to testing models via REST API - bin/nlpcraft.{sh|cmd} script provides a single centralized utility that makes a quick work out of all these routine tasks.

The script is only available in either official Apache source ZIP or in binary ZIP distributions. If you used Maven or similar build tools to add NLPCraft to your project as a dependency, this script will not be available out-of-the-box - you will need to download ZIP distribution separately.

ZIP Distribution

The bin/nlpcraft.{sh|cmd} script is only available in ZIP distributions. For Maven/Gradle/SBT projects you will need to download ZIP distribution separately.

The script is located in /bin sub-folder of the installation root and is available for Unix/Linux (nlpcraft.sh) and Windows (nlpcraft.cmd) environments. Both versions provide identical functionality.

Quick Start

The bin/nlpcraft.{sh|cmd} works in two modes:

  • Command line mode where it executes a single command at a time.
  • Interactive REPL mode. To enter REPL mode run bin/nlpcraft.sh without any arguments.

All the functionality and command syntax remain the same regardless of whether a command is executed from a command line or in REPL mode.

Script starts in the REPL mode by default. Note that in REPL mode the command prompt always shows REST server status, local probe status, REST access token (if any) as well as current working directory.

REPL Prompt

REPL prompt shows 4 pieces of information:

  • REST server status (ON if server is started and detected, OFF otherwise).
  • Local probe status (ON if probe is started and connected to the server, OFF otherwise).
  • REST access token if the user is signed in. This token is required for all REST calls.
  • Current user working directory.

Note that when using start-server command the default user account is automatically signed in upon server start and the access token is obtained. You can sign in with different user account, if necessary.

OS Commands

You can run any external OS-specific commands from bin/nlpcraft.{sh|cmd} script by prepending such command with a $ symbol. For example: 

            > $git pull

Runs git pull command in the current directory. 

            > $mvn clean package

Runs Maven build in the current directory. 

            > $cmd /c dir

Runs Windows dir command in the current directory using a separate shell.

Note that stdin, stderr and stdout of the new process will inherit from running bin/nlpcraft.{sh|cmd} script process. Running OS commands makes the most sense in REPL mode where you don't want to lose a session context while executing external OS commands. Technically, however, external OS commands can be executed from command line mode as well.

Usage

Commands supported by the bin/nlpcraft.{sh|cmd} script fall into 4 categories:

  1. Server Commands.

    Server commands allow to manage locally run REST server. Note that these commands only work for the REST server that runs on the same host as script. In other words, these commands cannot manage the remote REST server.

    REST server is a "fire-and-forget" component. During model development you will likely start the server only once and check its output from time to time as most of the work will be happening in the data probe that hosts your models.

    REPL Prompt

    REPL prompt shows 3 pieces of information:

    • REST server status (ON if server is started and detected, OFF otherwise).
    • REST access token if the user is signed in. This token is required for all REST calls.
    • Current user working directory.
  2. REST Commands:

    All REST commands provide convenient wrappers for REST API.

    Tha two main commands are call and rest - they provide generic REST call wrapper for both REPL and command line modes respectively. call command is convenient for the REPL mode, while rest command is more convenient for the command line mode.

    Following additional commands provide custom wrappers for frequently used REST APIs:

    REPL Wrapper REST API Help
    ask /ask/sync help --cmd=ask
    signin /signin help --cmd=signin
    signout /signout help --cmd=signout
    sugsyn /model/sugsyn help --cmd=sugsyn
