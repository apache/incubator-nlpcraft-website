Built-in Token Parser

{% scaladoc NCTokenParser NCTokenParser %} component implementation should parse user input plain text and split this text into tokens list. NLPCraft provides two English language token parser implementations. Also, project contains examples for French and Russia languages token parser implementations.

Apache OpenNLP Based Parser

There is {% scaladoc nlp/parsers/NCOpenNLPTokenParser NCOpenNLPTokenParser %} implementation. This implementation is wrapper on Apache OpenNLP project tokenizer.

Stanford NLP Based Parser

There is NCStanfordNLPTokenParser implementation. This implementation is wrapper on Stanford NLP project tokenizer.

Remarks

There are two different English language implementations are provided because they have some difference in their algorithms and can provide different list of tokens for same user text input. Some built-in components are required token parser instance as their parameter.

  • If you use Apache OpenNLP based components you should use Apache OpenNLP based parser in your model pipeline.
  • If you use Stanford NLP based components you should use Stanford based parser in your model pipeline.
